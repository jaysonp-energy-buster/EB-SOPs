# Energy Buster - Standard Operating Procedures

Central repository for all Energy Buster SOPs. This repo is the single source of truth for how we do things.

## Folder Structure

```
SOPs/
├── operations/        # Day-to-day operational procedures
├── onboarding/        # Team onboarding & training
├── engineering/       # Technical & development procedures
└── sales-marketing/   # Sales, marketing & GTM procedures
```

## How to Use

1. **Find an SOP** — Browse the relevant folder or search the repo.
2. **Create a new SOP** — Copy `TEMPLATE.md`, fill it in, and submit a PR.
3. **Update an existing SOP** — Create a branch, make your changes, and submit a PR for review.

## Naming Convention

Name SOP files as: `NNN-sop-name.md` (e.g., `001-client-onboarding.md`)

## Contributing

- All changes go through pull requests — no direct pushes to `main`.
- PRs require at least 1 approval before merging.
- Update the `last_reviewed` date in the SOP frontmatter when making changes.
